# Frontend Review Skill

Use this skill to review frontend code changes in this repository (Angular SPA, UI/UX, state, API integration).

## Scope

- Angular component/service behavior
- API integration and auth token flow
- Form validation and error handling
- Mobile/responsive behavior and accessibility basics
- UI consistency and functional usability

## Review approach

1. Focus on behavior, UX breakage risk, and integration correctness.
2. Check for real user-impacting problems before suggesting improvements.
3. Keep recommendations concise and prioritized.
4. Propose edits only when they improve reliability, usability, or clarity.

## Non-forcing rule (important)

- If the implementation already works well, do **not** invent changes just to "do something."
- "No changes needed" is a valid outcome.
- Do not force visual or stylistic tweaks unless explicitly requested.

## Output format

- Findings first (critical -> minor)
- Then optional polish ideas
- Then short final verdict: `changes required` or `no changes needed`
